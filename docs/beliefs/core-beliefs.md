## 불변 원칙 (모든 프로젝트 공통)

1. **리포 = 유일한 진실의 원천**
   - 리포에 없으면 에이전트에겐 존재하지 않음
   - 구두 합의, 채팅 스레드, 외부 문서의 결정사항은 리포에 기록해야 효력 발생

2. **점진적 공개**
   - 에이전트는 CLAUDE.md(맵)에서 출발하여 필요한 깊이만 탐색
   - 한 번에 모든 컨텍스트를 주입하지 않음

3. **기계적 강제 > 문서 설명**
   - 린터/CI로 잡을 수 있는 규칙은 문서에만 쓰지 말고 도구로 강제
   - 린터 에러 메시지에 수정 지침을 포함하여 에이전트가 자동 수정 가능하도록

4. **수정 지침 포함 에러**
   - 에러 메시지 작성 시 "무엇이 잘못되었는지" + "어떻게 고쳐야 하는지" 모두 포함
   - 에이전트가 에러 메시지만 보고 수정할 수 있어야 함

5. **계획은 일급 아티팩트**
   - 실행 계획을 docs/plans/에 커밋하고 진행 상태를 추적
   - 완료된 계획은 completed/로 이동하여 의사결정 히스토리 보존
