# Step 4: 리뷰

## 목적

구현된 코드를 **두 가지 관점**에서 검토합니다:
1. **Claude 자체 리뷰** -- 규칙 준수, 패턴 일관성, 비즈니스 로직 정합성
2. **Codex 리뷰** -- 로직 결함, 보안 취약점, 성능, 코드 품질

CRITICAL/HIGH 이슈가 모두 해소될 때까지 수정-재리뷰를 반복합니다 (최대 3회).

## 선행 조건

- [ ] Step 3 완료 (코드 변경이 존재)
- [ ] `context-chain.md`를 읽고 이전 결정사항을 확인함

> **단순 복잡도이고 사용자가 리뷰 스킵을 요청한 경우 Step 5로 건너뛸 수 있습니다.**

## 입력

- 변경된 코드 (diff)
- 분석 결과 (`01-analysis.md`)
- 승인된 계획 (`02-plan.md`)
- 개발 규칙 (`.harness/pipeline/rules.md`)
- 프로젝트 골든룰 (`docs/beliefs/golden-rules.md`)

## Pass 1: Claude 자체 리뷰

> 항상 실행. 규칙 기반 정적 검토.

### 체크리스트

#### A. 규칙 준수

- [ ] 보안 규칙 (SEC-*) 위반 없음
- [ ] 네이밍 규칙 (NAM-*) 준수
- [ ] 코딩 패턴 (COD-*) 준수
- [ ] 프레임워크 보호 (FRM-*) 위반 없음
- [ ] DB 규칙 (DB-*) 준수

#### B. 비즈니스 로직 정합성

- [ ] 요구사항(`01-analysis.md`)과 구현이 일치하는가
- [ ] 화면 흐름/버튼 동작이 올바르게 구현되었는가
- [ ] 계산 로직이 정확한가

#### C. 기존 코드 패턴 일관성

- [ ] 동일 모듈의 기존 패턴과 일치하는가
- [ ] 새로운 패턴이 불필요하게 도입되지 않았는가

#### D. 엣지케이스 및 안전성

- [ ] null/빈값 처리
- [ ] 배열/리스트 길이 불일치 처리
- [ ] 중복 데이터 방지
- [ ] 삭제 + 삽입 패턴에서 데이터 정합성

## Pass 2: Codex 리뷰

> Codex 플러그인 설치 시 실행. 미설치 시 Pass 1 결과만으로 진행.
> 설치: https://github.com/openai/codex-plugin-cc

### 실행 방법

변경된 파일 목록을 Codex에 전달하여 리뷰를 요청합니다:

```
Codex에게 아래 파일들의 코드 리뷰를 요청합니다:
- {변경 파일 1}
- {변경 파일 2}

리뷰 관점:
1. 로직 결함 (off-by-one, null 참조, 경쟁 조건 등)
2. 보안 취약점 (injection, 인증 우회, 민감 데이터 노출)
3. 성능 이슈 (N+1 쿼리, 불필요한 반복, 메모리 누수)
4. 코드 품질 (가독성, 중복, 복잡도)
```

### Codex 리뷰 초점

| 관점 | 설명 |
|------|------|
| 로직 결함 | off-by-one, null 참조, 경쟁 조건, 무한 루프, 타입 불일치 |
| 보안 취약점 | SQL injection, XSS, 인증 우회, 민감 데이터 노출 |
| 성능 | N+1 쿼리, 불필요한 반복, 대량 데이터 처리, 인덱스 미사용 |
| 코드 품질 | 가독성, 중복 코드, 과도한 복잡도, 네이밍 |

### Codex 사용 불가 시

Codex 플러그인이 설치되지 않은 환경에서는 Pass 1(Claude 자체 리뷰)의 결과만으로 리뷰를 진행합니다. 산출물에 "Codex 리뷰: N/A (미설치)"로 기록합니다.

## 리뷰 프로세스

1. **Pass 1 실행** -- Claude 자체 리뷰 수행, 결과 기록
2. **Pass 2 실행** -- Codex 리뷰 요청, 결과 기록 (미설치 시 스킵)
3. **결과 통합** -- 두 리뷰의 발견 사항을 통합하여 종합 판정
4. CRITICAL/HIGH 이슈가 있으면:
   - 사용자에게 보고
   - 수정 방향 확인 후 코드 수정
   - 재리뷰 (최대 3회 반복)
5. 3회 후에도 미해소 -> 사용자에게 알림
6. 모든 CRITICAL/HIGH 해소 -> 종합 판정: PASS
